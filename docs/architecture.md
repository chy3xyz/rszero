# rszero 架构设计

> 本文档详细描述 rszero 的系统架构、模块设计和数据流。

---

## 1. 整体架构

rszero 采用三层架构设计，完全对齐 go-zero 的「API 网关 + RPC 微服务 + 基础设施」模式：

```
┌──────────────────────────────────────────────────────────────┐
│                      接入层 (Access Layer)                    │
│  ┌─────────────────┐  ┌─────────────────┐  ┌──────────────┐ │
│  │   REST Gateway  │  │   gRPC Gateway  │  │  WebSocket   │ │
│  │   (Axum 0.7)   │  │   (Volo 0.10)   │  │   (planned)  │ │
│  └────────┬────────┘  └────────┬────────┘  └──────────────┘ │
└───────────┼────────────────────┼────────────────────────────┘
            │                    │
┌───────────▼────────────────────▼────────────────────────────┐
│                    服务层 (Service Layer)                     │
│  ┌─────────────────┐  ┌─────────────────┐  ┌──────────────┐ │
│  │  User Service   │  │  Order Service  │  │  Pay Service │ │
│  │  (REST + RPC)   │  │  (REST + RPC)   │  │  (RPC only)  │ │
│  └────────┬────────┘  └────────┬────────┘  └──────────────┘ │
└───────────┼────────────────────┼────────────────────────────┘
            │                    │
┌───────────▼────────────────────▼────────────────────────────┐
│                  基础设施层 (Infrastructure)                   │
│  ┌──────────┐ ┌──────┐ ┌───────┐ ┌──────┐ ┌──────────────┐  │
│  │ Config   │ │ Log  │ │ Cache │ │  DB  │ │   Queue      │  │
│  │(figment) │ │(trace│ │(fred) │ │(sqlx)│ │  (lapin)     │  │
│  └──────────┘ └──────┘ └───────┘ └──────┘ └──────────────┘  │
│  ┌──────────┐ ┌──────┐ ┌───────┐ ┌──────┐ ┌──────────────┐  │
│  │ Limit    │ │Break │ │Discov │ │Trace │ │   Metrics    │  │
│  │(governor)│ │(cb)  │ │(etcd) │ │(otel)│ │ (prometheus) │  │
│  └──────────┘ └──────┘ └───────┘ └──────┘ └──────────────┘  │
└──────────────────────────────────────────────────────────────┘
```

## 2. 模块设计

### 2.1 核心模块

| 模块 | 职责 | 依赖 | 状态 |
|------|------|------|------|
| `rest` | HTTP 服务器、路由、Handler | axum 0.7 | ✅ |
| `rpc` | RPC 客户端/服务器 | volo 0.10 | 🚧 |
| `config` | 配置加载、热重载 | figment, dotenvy | ✅ |
| `log` | 结构化日志 | tracing | ✅ |
| `cache` | Redis + 内存缓存 | fred 6.0, dashmap | ✅ |
| `queue` | 消息队列 | lapin | 🚧 |
| `store` | 数据库连接、迁移 | sqlx, sea-orm | 🚧 |

### 2.2 治理模块

| 模块 | 职责 | 依赖 | 状态 |
|------|------|------|------|
| `limit` | 全局限流 | tower-governor | ✅ |
| `breaker` | 熔断降级 | 内置 | ✅ |
| `discovery` | 服务注册发现 | etcd-client | ✅ |
| `shedder` | 负载脱落 | 内置 | ✅ |
| `timeout` | 超时控制 | tokio | ✅ |
| `retry` | 指数退避重试 | 内置 | ✅ |

### 2.3 横切模块

| 模块 | 职责 | 依赖 | 状态 |
|------|------|------|------|
| `error` | 统一错误处理 | thiserror | ✅ |
| `middleware` | JWT/日志/追踪/验证 | jsonwebtoken, axum | ✅ |
| `trace` | 分布式追踪 | opentelemetry | 🚧 |
| `health` | 健康检查 | 内置 | ✅ |
| `metrics` | Prometheus 指标 | 内置 | ✅ |
| `openapi` | OpenAPI 文档生成 | serde, serde_yaml | ✅ |
| `concurrent` | MapReduce/函数式流 | tokio | ✅ |

## 3. 数据流

### 3.1 请求处理流程

```
Client Request
    │
    ▼
┌─────────────────────┐
│  Load Shedder       │ ← 系统过载时拒绝请求
└─────────┬───────────┘
          ▼
┌─────────────────────┐
│  Request ID         │ ← 生成/传播请求 ID
└─────────┬───────────┘
          ▼
┌─────────────────────┐
│  Trace Middleware   │ ← 分布式追踪
└─────────┬───────────┘
          ▼
┌─────────────────────┐
│  Validation         │ ← 请求体校验
└─────────┬───────────┘
          ▼
┌─────────────────────┐
│  Rate Limiter       │ ← 限流
└─────────┬───────────┘
          ▼
┌─────────────────────┐
│  JWT Auth           │ ← 认证（可选）
└─────────┬───────────┘
          ▼
┌─────────────────────┐
│  Handler            │ ← 业务逻辑
│  ├─ Cache Check     │
│  ├─ Circuit Breaker │
│  ├─ DB Query        │
│  └─ Cache Write     │
└─────────┬───────────┘
          ▼
┌─────────────────────┐
│  JsonResponse       │ ← 统一响应格式
└─────────────────────┘
```

### 3.2 缓存策略

```
Request
    │
    ▼
┌──────────────┐     Hit    ┌──────────┐
│  Cache Get   │───────────▶│  Return  │
└──────┬───────┘            └──────────┘
       │ Miss
       ▼
┌──────────────┐     OK      ┌──────────┐
│  DB Query    │────────────▶│  Cache   │
└──────┬───────┘             │  Set+TTL │
       │ Error               └──────────┘
       ▼
┌──────────────┐
│  Error Resp  │
└──────────────┘
```

## 4. 配置体系

```
.rszero/
├── .env                    # 环境变量（开发）
├── .env.production         # 环境变量（生产）
├── etc/
│   ├── api.yaml            # API 网关配置
│   └── user-rpc.yaml       # RPC 服务配置
└── migrations/
    ├── 001_create_users.sql
    └── 002_create_orders.sql
```

配置加载优先级：
1. 环境变量 (`RSZERO_*` 前缀)
2. YAML 配置文件
3. `.env` 文件
4. 默认值

## 5. 错误处理体系

```
RszeroError (统一错误类型)
├── Config          ← 配置错误
├── Database        ← 数据库错误
├── Cache           ← 缓存错误
├── Rpc             ← RPC 调用错误
├── Http            ← HTTP 错误（带状态码）
├── Auth            ← 认证错误
├── RateLimit       ← 限流错误
├── CircuitBreaker  ← 熔断错误
├── NotFound        ← 资源不存在
├── Discovery       ← 服务发现错误
├── Queue           ← 消息队列错误
├── Serialization   ← 序列化错误
└── Internal        ← 内部错误
```
